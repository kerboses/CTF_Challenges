### Command Injection Demonstration (Lab Environment)

1. Open a web browser and navigate to the following URL:

   ```
   http://15.135.158.171/dir/
   ```

2. Locate the input field on the webpage where user input can be entered.

   <img width="1001" height="55" alt="Screenshot from 2026-07-01 20-26-56" src="https://github.com/user-attachments/assets/665910b1-8de5-42e6-a043-6922d1acd6e4" />


3. Observe that the application sends the input value through the **HTTP GET** method. The entered value is appended to the URL as a query parameter.

4. Certain special characters and ASCII symbols cannot be transmitted directly in a URL because they have reserved meanings or are not processed correctly by the web server.


<img width="1130" height="455" alt="Screenshot from 2026-07-01 20-31-39" src="https://github.com/user-attachments/assets/5cc57624-5da7-47fa-a8bd-f99188a23f19" />

5. To safely include special characters in the URL, convert them into **URL-encoded** format using an online URL encoder, such as:

   ```
   https://meyerweb.com/eric/tools/dencoder/
   ```
<img width="287" height="52" alt="Screenshot from 2026-07-01 20-32-19" src="https://github.com/user-attachments/assets/b5974781-4ce5-4204-93ec-2a4007a049ec" />

6. Copy the URL-encoded output generated by the encoder.

   <img width="754" height="89" alt="Screenshot from 2026-07-01 20-32-45" src="https://github.com/user-attachments/assets/21824e00-fa3a-46b2-afa9-459af3fd8e2b" />


7. Paste the encoded value into the appropriate parameter in the browser's address bar and submit the request.

<img width="1676" height="149" alt="Screenshot from 2026-07-01 20-38-57" src="https://github.com/user-attachments/assets/8c1ab1dd-e870-46cd-bf6f-9a7d4f1f4334" />

8. Observe how the web application processes the encoded input and note the server's response. This helps demonstrate how web applications handle URL-encoded data transmitted through the GET method.
